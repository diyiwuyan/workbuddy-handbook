**WorkBuddy** 是腾讯最新推出的全场景职场 AI 智能体工作台，

面向 人力资源、行政、运营、销售、研发等不同职场角色，是一款能够像真正同事一样思考、执行任务并交付结果的 AI 办公应用。

## WorkBuddy 产品定位

### 它到底是什么

WorkBuddy 是腾讯推出的**全场景 AI Agent 桌面工作台**。

很多人第一次听到 WorkBuddy，会下意识把它理解成"又一个 AI 聊天工具"。

但如果只把 WorkBuddy 当成"一个能对话的 AI"，你很容易低估它。

WorkBuddy 真正重要的地方，不是它能回答问题，而是它代表了 AI 助手的一次角色变化：

> **以前，AI 是对话框里回答你问题的人。**
> **后来，AI 是在编辑器里和你一起改代码的人。**
> **现在，WorkBuddy 是一个可以被交代任务的工程执行者——而且它能操作你电脑上的真实文件。**

它不只是回答你"这篇文章怎么写"，而是可以：
- 读取你本地的文件和项目
- 理解上下文和需求
- 制定执行计划
- 调用 Skill 完成专项任务
- 连接外部工具（MCP）扩展能力
- 记住你的偏好和历史
- 把结果直接交付到你需要的地方

### 核心能力一览

| 能力 | 说明 |
|---|---|
| **一句话任务指派** | 用自然语言描述需求，无需复杂操作 |
| **自主规划与执行** | 自动拆解任务、规划步骤、交付结果 |
| **并行多任务** | 同时启动和管理多个任务，通过侧边栏切换 |
| **本地文件操作** | 读取授权文件夹，批量处理文件 |
| **可验证的结果** | 生成的文档、表格、PPT 可直接使用 |
| **Skill 扩展** | 兼容 OpenClaw 技能生态，7万+ 社区技能 |
| **专家系统** | 100+ 领域专家，按需召唤 |
| **自动化** | 定时触发，724 小时无人值守执行 |
| **连接器** | 打通腾讯文档、QQ 邮箱、飞书等外部服务 |

### 三大工作模式

WorkBuddy 的核心交互分为三个模式，覆盖不同信任程度和任务复杂度：

| 模式 | 权限 | 行为 | 打个比方 |
|---|---|---|---|
| **Ask** | 仅对话 | 只回答问题，完全不碰文件 | 只提建议的顾问 |
| **Plan** | 先读后写 | 先出执行计划，逐条确认后才执行 | 写好计划书再动手的助手 |
| **Craft** | 可读写 + 执行 | 直接操作文件、运行代码，每步仍需你确认 | 坐你旁边干活的搭档 |

**使用建议**：
- 日常 90% 的任务用 **Plan**（先计划后执行，可控）
- 咨询讨论用 **Ask**（不需要操作文件）
- 批量文件处理和写代码用 **Craft**（效率高，但需盯紧）

## WorkBuddy 与同类工具的区别

### vs ChatGPT

| 对比维度 | ChatGPT | WorkBuddy |
|---|---|---|
| 主要定位 | 对话式 AI 助手 | AI Agent 桌面工作台 |
| 文件操作 | 需手动上传/下载 | 直接读写本地授权文件 |
| 外部工具 | 插件/GPTs | MCP 连接器（更开放） |
| 自动化 | 需手动触发 | 支持定时自动化 |
| 专属专家 | 不支持 | 100+ 领域专家，按需召唤 |

**更合理的用法**：用 ChatGPT 做头脑风暴、想法验证；用 WorkBuddy 执行完整的工作流和任务交付。

### vs Cursor / Claude Code

| 对比维度 | Cursor / Claude Code | WorkBuddy |
|---|---|---|
| 主要定位 | AI 编辑器 / 终端 Agent | 全场景桌面 Agent |
| 适用场景 | 代码开发 | 办公 + 开发 + 内容创作 |
| 文件操作 | 编辑器内 | 本地文件系统 + 终端 |
| 内容创作 | 不支持 | 支持（文档、PPT、PDF） |
| 专家系统 | 不支持 | 支持（可召唤领域专家） |
| 自动化 | 不支持 | 支持（定时任务） |

**合理的用法是组合使用**：用 Cursor 做日常编码；用 WorkBuddy 做跨工具全流程任务和自动化。
